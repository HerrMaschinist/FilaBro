## Ziel und Nutzen der App
FilaBro ist eine App zur Verwaltung von 3D Druck Filamentspulen. Die App verbindet sich mit einem Spoolman Server in meinem Netzwerk und zeigt Spulen, Filamente, Hersteller, Restgewicht und Favoriten. Kernnutzen ist schnelles Finden und Aktualisieren von Spulen per Barcode, QR Code, NFC und OCR, plus Offline Nutzung über lokalen Cache.

## Zielgruppe
Ich und meine Freunde, die 3D Druck betreiben. Es muss sich wie ein richtiges Produkt anfühlen. Modern, schnell, stabil, übersichtlich.

## Spoolman Server und API
Standard Server URL ist:
http://192.168.50.10:7912

Health Check:
GET /api/v1/health

Spulen Liste:
GET /api/v1/spool mit expandierten Filament Daten
Wenn expand Query Parameter unterstützt wird, nutze:
?expand[]=filament

Spule Details:
GET /api/v1/spool/{id}

Spule aktualisieren:
PATCH /api/v1/spool/{id}
Beispiel Feld:
remaining_weight

Wichtig: Die App muss HTTP im lokalen Netzwerk unterstützen. Setze Cleartext HTTP in Android Konfiguration korrekt um.

## Tech Stack und Rahmen
Du baust die App als Expo React Native Mobile App, damit sie schnell testbar ist, aber so strukturiert, dass ich später jederzeit auf ein natives Setup wechseln oder ein Custom Dev Client bauen kann.

Anforderungen an Technik
1. TypeScript verwenden.
2. React Navigation für Navigation.
3. State Management stabil und simpel, zum Beispiel Zustand oder Redux Toolkit.
4. Networking über eine stabile Lösung, zum Beispiel fetch plus TanStack Query für Cache und Requests.
5. Lokale Datenbank: SQLite in Expo, mit sauberem Schema und Migrationen.
6. Secure Storage: sensible Settings wie Server URL sicher speichern.
7. Hintergrund Sync: Änderungen in Queue speichern und später synchronisieren.

Wichtig: Barcode, Kamera, NFC und OCR sind Pflicht Features.
Wenn bestimmte Features nicht in Expo Go laufen, dann implementiere es so, dass ein Custom Dev Client oder Prebuild möglich ist. Ziel ist, dass es später auf einem echten Android Gerät zuverlässig läuft.

## Features im Detail
### Feature 1: Onboarding und Verbindungsmanagement
Beim ersten Start:
1. Eingabemaske für Server URL mit Default http://192.168.50.10:7912
2. Health Check Button ruft GET /api/v1/health auf.
3. Bei Erfolg speichern in Secure Storage.
4. Bei Fehlern klare Meldung, mit Retry und Hinweis.

### Feature 2: Hauptnavigation
Bottom Navigation mit Tabs:
1. Spulen
2. Favoriten
3. Scanner
4. Einstellungen

### Feature 3: Spulen Liste
Datenquelle:
1. Online: GET /api/v1/spool mit expand filament falls möglich.
2. Offline: aus lokalem Cache.

Anzeige pro Spule:
1. Farbindikator
2. Filament Name
3. Hersteller
4. Material
5. Restgewicht als Zahl und Fortschrittsbalken

Funktionen:
1. Suche
2. Filter: Hersteller, Material, Favoriten, Restgewicht Bereich
3. Sortierung: Name, Restgewicht, Hersteller, Material
4. Pull to Refresh
5. Sync Status Icon

### Feature 4: Favoriten
Favoriten Status wird lokal gespeichert.
Favoriten Liste zeigt nur Favoriten.
Favoriten Toggle in Liste und Detail.

### Feature 5: Spule Detailansicht
Beim Öffnen:
GET /api/v1/spool/{id}
Zeige alle relevanten Daten.

Gewichtsanpassung Modi:
1. Slider von 0 bis Startgewicht
2. Manuelle Eingabe
3. Waagenmodus
   Nutzer gibt Gesamtgewicht ein
   Restgewicht wird berechnet als Gesamtgewicht minus Leergewicht
   Leergewicht kommt aus Spool Daten wenn vorhanden

Speichern:
PATCH /api/v1/spool/{id} mit remaining_weight
Wenn offline: speichere Änderung in pending queue und markiere als ausstehend.

Weitere Aktionen:
1. QR Code anzeigen für Spulen URL
2. Teilen Funktion
3. NFC Tag schreiben mit URL zur Spule
4. Favorit umschalten

### Feature 6: Scanner
Drei Modi:
1. Barcode oder QR Code Scan über Kamera
2. NFC Scan und NFC Write
3. OCR Scan

Barcode oder QR Ablauf:
1. Scanne Code
2. Extrahiere Spulen ID oder URL
3. Öffne Detailansicht oder Trefferliste

NFC Ablauf:
1. NFC lesen erkennt URL oder ID
2. Öffne Detailansicht
3. NFC schreiben in Detailansicht: schreibe URL auf Tag

OCR Ablauf:
1. Kamera Aufnahme
2. Text Recognition auf dem Gerät
3. Extrahiere Hersteller, Material, Farbe, Gewicht wenn möglich mit Regex
4. Suche in lokalem Cache nach passenden Kandidaten
5. Zeige Trefferliste, Nutzer bestätigt

Wichtig: OCR und NFC müssen so integriert sein, dass es auf echten Android Geräten funktioniert. Wenn Expo Go nicht reicht, nutze Custom Dev Client Weg und dokumentiere es.

### Feature 7: Offline First und Sync Engine
Lokale Datenbank Tabellen:
1. spool_cache
2. filament_cache
3. vendors_cache
4. favorites
5. pending_updates
6. optional scan_history

Sync Regeln:
1. Wenn online: cache aktualisieren
2. Wenn offline: app funktioniert aus cache
3. pending_updates werden in Reihenfolge synchronisiert sobald online
4. Konflikte: wenn server Werte neuer sind, dann server gewinnt oder last write wins, dokumentiere Entscheidung klar

### Feature 8: Einstellungen
Einstellungen Screen:
1. Server URL ändern und testen
2. Theme: auto, hell, dunkel
3. Standard Gewichtsanpassung: slider, manuell, waage
4. OCR Sprache falls relevant
5. Export und Backup

6. Debug Bereich

   zeige letzten sync status
   zeige queue länge
